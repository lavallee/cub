# Cookbook

Complete workflows showing Cub in action. Each recipe walks through a real use case from start to finish, with the exact commands you would run. Pick the one closest to what you are trying to do and follow along.

<div class="feature-grid" markdown>

<div class="feature-card" markdown>

#### [Bug Fix to PR](bug-fix-to-pr.md)

Fix a bug from initial report through to a merged pull request. Covers task creation, single-shot execution, review, and PR workflow.

</div>

<div class="feature-card" markdown>

#### [Feature from Spec](feature-from-spec.md)

Build a feature through the full planning pipeline: spec, orient, architect, itemize, stage, run, and ship.

</div>

<div class="feature-card" markdown>

#### [Overnight Batch](overnight-batch.md)

Run autonomous agents overnight on a large task queue. Set budgets, monitor progress, and review results in the morning.

</div>

<div class="feature-card" markdown>

#### [Direct Session Tracking](direct-session-tracking.md)

Track work done in interactive Claude Code sessions using symbiotic hooks. Claim tasks, log progress, and build ledger entries without `cub run`.

</div>

<div class="feature-card" markdown>

#### [Add to Existing Project](existing-project.md)

Set up Cub in an existing codebase. Initialize configuration, import existing work, and run your first autonomous task.

</div>

<div class="feature-card" markdown>

#### [Parallel Epics](parallel-epics.md)

Execute multiple epics in parallel using git worktrees. Isolate work streams and merge results independently.

</div>

</div>

## Prerequisites

These recipes assume you are familiar with [Core Concepts](../getting-started/concepts.md) and have completed the [Quick Start](../getting-started/quickstart.md). You should have Cub installed, at least one AI harness available, and a project initialized with `cub init`.
